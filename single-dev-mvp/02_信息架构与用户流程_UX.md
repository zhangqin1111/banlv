## 1. 信息架构

```text
App
├─ Splash
├─ Main Tabs
│  ├─ Home
│  │  ├─ 解忧树洞
│  │  ├─ 情绪气象台
│  │  ├─ 解忧盲盒 Lite
│  │  └─ 我的 momo
│  ├─ 今日情绪
│  └─ 我的
├─ Treehole Chat
├─ Mood Weather
│  ├─ Emotion Type
│  ├─ Intensity
│  ├─ One-line Note
│  └─ Invite Cards
├─ Modes
│  ├─ 光粒漂流
│  ├─ 云团呼吸
│  └─ 气泡释放

   ├─ Privacy
   ├─ Delete
   ├─ Report
```

---

## 2. 全局导航规则

- 底部 3 个一级 Tab：
  - 首页
  - 今日情绪
  - 我的
- 树洞、情绪气象台、三模式、盲盒、成长页都是二级流程页
- 二级页返回时默认回到来源页
- 高危熔断页优先级最高，可打断树洞和签到后的普通流转

---

## 3. 核心流程

### 3.1 主流程 A：从首页进入情绪气象台
1. 首页
2. 情绪气象台
3. 用户完成情绪输入
4. 系统给出一条共情短文
5. 出现 3 张邀请卡：
   - 聊一聊
   - 推荐情绪模式
   - 抽个盲盒
6. 用户自主选择去向
7. 完成后进入结束反馈
8. 保存到记录

### 3.2 主流程 B：直接树洞
1. 首页或首页快捷入口
2. 树洞页
3. 对话 3-5 轮
4. 插入去模式或盲盒的温柔建议
5. 结束反馈

### 3.3 主流程 C：直接抽盲盒
1. 首页
2. 盲盒 Lite
3. 抽取安慰卡
4. 可收藏
5. 返回首页或记录

### 3.4 主流程 D：成长反馈
1. 首页
2. 我的 `momo`
3. 查看阶段、成长值、最近变化

## 4. 状态机

### 4.1 情绪气象台状态机

```text
idle
-> emotion_selected
-> intensity_selected
-> optional_note_filled
-> submitted
-> empathy_shown
-> choices_shown
-> next_module_entered
```

规则：
- 一句话描述可为空
- 提交后不自动跳转
- 推荐卡按当前情绪类型变化

### 4.2 树洞状态机

```text
idle
-> user_input
-> sending
-> ai_streaming
-> ai_done
-> optional_suggestion_shown
-> feedback_pending
-> feedback_done
```

异常分支：
- network_error
- timeout_fallback
- safety_block

### 4.3 模式状态机

```text
intro
-> active
-> complete
-> feedback_pending
-> feedback_done
```

所有模式共用规则：
- 无失败态
- 中途退出也记一次尝试

### 4.4 盲盒状态机

```text
idle
-> draw_start
-> card_revealed
-> saved_or_closed
```

### 4.5 高危熔断状态机

```text
message_submit
-> high_risk_detected
-> normal_flow_stopped
-> safety_panel_shown
-> user_choose_exit
```

出口：
- 我想自己待会
- 我想看安全建议
- 我需要求助信息

## 5. 页面交互细节

### 5.1 首页
#### 目标
保留原始需求的“治愈空间感 + 模块入口感”。

#### 必备元素
- `momo` 主视觉
- 小岛或漂浮空间背景
- 四个核心入口
- 最近一次记录摘要

#### 行为
- 点击树洞进入聊天
- 点击气象台进入签到
- 点击盲盒进入抽卡
- 点击我的 `momo` 进入成长页

### 5.2 情绪气象台
#### 布局顺序
1. 标题和一句提问
2. 情绪类型卡片
3. 强度滑杆
4. 一句话输入
5. 提交按钮
6. 提交后共情文案
7. 3 张邀请卡

#### 交互要求
- 情绪卡片必须大、可点
- 强度滑杆默认在中间
- 提交后页面不跳闪
- 邀请卡文案必须柔和

### 5.3 树洞页
#### 布局
- 顶部：返回 + 标题 + `momo` 小状态
- 中部：对话流
- 底部：输入框 + 发送按钮

#### 对话要求
- 回复不宜过长
- 3 轮后允许出现“想不想试试对应模式”的建议卡
- 用户随时可以返回

### 5.4 光粒漂流
#### 目标
明亮、轻盈、上升感。

#### 布局
- 中央：`momo`
- 周围：可触碰光粒
- 顶部：极简引导
- 底部：结束按钮

### 5.5 云团呼吸
#### 目标
缓慢、柔和、接纳。

#### 布局
- 中央：`momo`
- 外圈：呼吸环
- 下方：呼吸引导

### 5.6 气泡释放
#### 目标
安全释放，不暴力。

#### 布局
- 中央：`momo`
- 前景：可点击气泡或可拖散雾层
- 收尾：慢呼吸提示

### 5.7 盲盒 Lite
#### 布局
- 上方：烦恼一句话输入，可选
- 中央：盲盒按钮
- 下方：卡片展示区

#### 要求
- 不做复杂动画
- 抽卡后允许收藏

### 5.8 我的 `momo`
#### 布局
- `momo` 当前形态
- 当前阶段
- 成长值
- 最近一次成长来源

## 6. 空态与错态

### 6.1 首页空态
- 文案：“今天想从哪里开始都可以。”

### 6.2 今日情绪空态
- 文案：“第一条记录，留给现在的你。”

### 6.3 树洞超时
- 文案：“我还在整理想说的话，我们换个更短的说法试试。”

### 6.4 模式退出
- 文案：“已经够了，停在这里也可以。”

### 6.5 盲盒空态
- 文案：“想不想给今天抽一张温柔的小卡片？”

### 6.6 安全熔断
- 文案不使用“危险”“异常”“失败”

---

## 7. 文案规范

### 7.1 推荐语气
- 温柔
- 邀请式
- 共情
- 不评判
- 不说教

### 7.2 禁用表达
- 通关
- 胜利
- 任务完成
- 你应该
- 系统检测到异常
- 你已经恢复

### 7.3 推荐表达
- 我在
- 可以慢一点
- 如果你愿意
- 先不用急着说清楚
- 这一刻也算被接住了

---

## 8. 交互底线

- 不自动强制进入模式
- 不让用户失败
- 不用红字制造压力
- 不让 `momo` 说出依赖型挽留文案
- 不使用“连续登录奖励”驱动核心留存
