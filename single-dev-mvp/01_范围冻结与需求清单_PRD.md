# EmoBot 原始需求兼容版 PRD

## 1. 产品定义

EmoBot 首版是一个 **治愈系 AI 情绪陪伴 App**，但与极简版不同，这版必须保留原始需求中的三块差异化骨架：

1. 情绪气象台
2. 三种情绪模式
3. `momo` 成长反馈

并在单人可落地的条件下增加：
- 解忧盲盒 Lite
- 我的 `momo`

---

## 2. 范围冻结

### 2.1 P0 功能
1. Splash
2. 游客进入
3. 首页治愈场景 + `momo`
4. 解忧树洞
5. 情绪气象台
6. 快乐模式：光粒漂流
7. 低落模式：云团呼吸
8. 愤怒模式：气泡释放
9. 解忧盲盒 Lite
10. 我的 `momo`
11. 最近 7 天回看
12. 高危熔断
13. 删除 / 举报 / 隐私 / 年龄说明

### 2.2 P1 功能
- 首页“上次的你”
- 今日情绪摘要卡
- 趋势迷你图
- 盲盒收藏页
- 会话摘要卡
- 游客转昵称资料

### 2.3 本阶段明确不做
- 99 关
- 关卡制和竞技化表达
- 专注森林
- 社区
- 星座 / 塔罗 / MBTI
- 语音输入输出
- 小程序 / Web
- 多端同步
- 长期记忆 / 向量库
- 商城 / 皮肤系统

---

## 3. 目标用户

### 3.1 核心用户
- 18-35 岁
- 情绪波动明显，但不期待医疗服务
- 希望在低压力状态下被陪伴、被理解
- 对可爱角色和轻互动有接受度

### 3.2 非目标用户
- 未成年人公开版用户
- 高危重症用户
- 需要正式心理治疗的用户

---

## 4. 核心用户故事

### 4.1 用户故事 A：今天状态不一样
作为一个今天情绪和昨天不太一样的人，  
我希望 EmoBot 能给我和今天状态匹配的互动方式，  
而不是无论我快乐还是生气都只有同一种反馈。

### 4.2 用户故事 B：不想长聊
作为一个现在不想说太多的人，  
我希望签到后可以直接进入一个短而轻的互动场景，  
这样我也能得到一些情绪缓和。

### 4.3 用户故事 C：想拿一点安慰
作为一个不想长聊也不想做模式互动的人，  
我希望能抽一张温柔的小卡片，  
这样我也有一个轻量出口。

### 4.4 用户故事 D：想看到 `momo` 在陪我成长
作为一个反复回来使用的人，  
我希望 `momo` 的状态会因为我的使用而变化，  
这样我会更愿意回来。

---

## 5. 页面目标

| 页面 | 目标 | P0 验收 |
|---|---|---|
| Splash | 建立品牌与角色感 | 2 秒内进入首页 |
| 首页 | 建立世界感和入口分发 | 可进四大模块 |
| 情绪气象台 | 完成一次情绪记录并获得邀请式去向 | 可提交并显示三种去向 |
| 树洞页 | 完成一次有效陪伴对话 | 支持至少 3 轮文本对话 |
| 光粒漂流 | 放大愉悦感 | 45-60 秒可走通 |
| 云团呼吸 | 缓和低落 | 60-90 秒可走通 |
| 气泡释放 | 安全释放情绪 | 45-60 秒可走通 |
| 盲盒 Lite | 给出轻安慰卡 | 可抽取并收藏 |
| 我的 `momo` | 展示成长状态 | 可看到阶段和成长值 |
| 今日情绪 / 记录页 | 查看最近 7 天记录 | 可看列表和摘要 |
| 设置页 | 合规和控制 | 删除 / 举报 / 隐私 / 年龄可用 |

## 6. 模块需求

### 6.1 Splash
#### 目标
让用户一打开就看到治愈感和 `momo`。

#### P0 内容
- Logo
- `momo` 漂浮
- 轻呼吸背景

### 6.2 游客进入
#### 目标
降低门槛。

#### 规则
- 首次打开自动生成匿名设备身份
- 不强制手机号
- 不强制昵称

### 6.3 首页
#### 目标
兼容原始 PDF 的“场景感 + 多入口”设计，但做极简版。

#### 内容
- `momo`
- 小岛或漂浮空间感背景
- 四个入口：
  - 解忧树洞
  - 情绪气象台
  - 解忧盲盒 Lite
  - 我的 `momo`
- 最近一次记录摘要

### 6.4 情绪气象台
#### 目标
保留原始模块核心。

#### 输入项
- 情绪类型
- 强度 0-10
- 一句话描述，可选

#### 输出
- 一句共情短文
- 三种邀请式卡片：
  - 聊一聊
  - 推荐的情绪模式
  - 抽个盲盒

#### 规则
- 不自动强制进入模式
- 仍然必须由用户选择

### 6.5 解忧树洞
#### 目标
完成陪伴价值。

#### 核心要求
- 文本对话
- 回复温柔、简短、可继续
- 3 轮后可建议去对应模式
- 高危时必须熔断

### 6.6 快乐模式：光粒漂流
#### 目标
保留原始“快乐情绪放大”精神。

#### 玩法
- 点击 / 滑动收集光粒
- `momo` 触碰后发光
- 结束给出一句收藏式文案

### 6.7 低落模式：云团呼吸
#### 目标
保留原始“低落接纳与缓和”精神。

#### 玩法
- 3 轮呼吸引导
- 节奏缓慢
- 无失败态

### 6.8 愤怒模式：气泡释放
#### 目标
保留原始“安全释放”精神。

#### 玩法
- 点击戳破气泡或拖散红雾
- 最后收尾一轮慢呼吸
- 不使用暴力化文案

### 6.9 解忧盲盒 Lite
#### 目标
保留原始盲盒模块，但显著降级。

#### 卡片类型
- 温柔提醒
- 小行动建议
- 小确幸
- 稳定呼吸提示

#### 行为
- 可抽 1 张
- 可收藏

### 6.10 我的 `momo`
#### 目标
保留成长反馈。

#### 内容
- 当前阶段
- 成长值
- 最近一次互动带来的变化

#### 阶段
- 幼体
- 进化体
- 发光体

### 6.11 记录回看
#### 目标
展示近 7 天记录。

#### 内容
- 日期
- 情绪
- 强度
- 摘要
- 使用过的模块

### 6.12 高危熔断
#### 目标
避免危险自由陪聊。

#### 触发
- 自伤
- 自杀
- 他伤
- 家暴
- 极端绝望表达

#### 结果
- 停止普通聊天
- 进入安全链路

### 6.13 删除 / 举报 / 隐私 / 年龄说明
#### 目标
满足最小合规底线。

---

## 7. 非功能需求

### 7.1 体验
- 主要流程单手可用
- 页面过渡柔和
- 用户在 2 分钟内能完成一次签到 + 进入一个模块

### 7.2 性能
- 首页首屏 <= 2.5 秒
- 对话首 token 尽量 <= 2 秒
- 三种模式在中端机不明显掉帧

### 7.3 安全
- 高危可熔断
- 日志不记录完整高敏文本
- 举报与删除真实可用

---

## 8. 非目标

这版仍然不解决：
- 完整游戏体验
- 长期关系型 AI 陪伴
- 商业化闭环
- 社区治理
- 内容宇宙扩张

---

## 9. 上线判定

满足以下条件才算 MVP 完成：
- P0 功能全部上线
- 三种模式全部可走通
- 盲盒 Lite 可用
- 我的 `momo` 可展示成长
- 高危熔断通过 QA
- 删除 / 举报 / 隐私 / 年龄说明可用

---

## 10. 验收定义

### 10.1 产品层面
- 用户能完成一次签到
- 用户能进入树洞、模式或盲盒任一出口
- 用户能看到 `momo` 成长反馈
- 用户能在记录页看到本次使用

### 10.2 风险层面
- 高危场景不继续自由聊天
- 模式不会被设计成失败惩罚或竞技系统

### 10.3 开发层面
- 前后端字段一致
- 关键错误态可控
- 埋点和日志存在

---

## 11. 砍需求规则

如果进度落后，按以下顺序砍：
1. 先砍趋势图
2. 再砍首页最近摘要的丰富样式
3. 再砍盲盒收藏页
4. 再砍“上次的你”
5. 不砍三种模式
6. 不砍高危熔断
7. 不砍删除 / 举报 / 隐私
