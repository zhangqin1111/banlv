# EmoBot 开发文档包索引

## 1. 目标

这套文档包用于指导 **1 名开发者在 8 周主开发 + 2 周缓冲内完成 EmoBot 原始需求兼容版 MVP**。

产品定义已经冻结为：

**治愈系 AI 情绪陪伴 App + 情绪气象台 + 三种情绪微互动 + 解忧盲盒 Lite + 极简 `momo` 成长**

不是原始完整版，但也不再是极简陪伴工具版。

## 4. 开发时的理解顺序

### 4.1 先理解“恢复了什么”
这版相较上一版，明确恢复：
- 情绪气象台
- 三种情绪模式
- 解忧盲盒 Lite
- 我的 `momo`

### 4.2 再理解“仍然砍掉什么”
仍然不做：
- 99 关
- 专注森林
- 社区
- 塔罗 / MBTI / 星座
- 语音
- 多端同步

### 4.3 最后理解“为什么还能单人做”
因为恢复的是：
- 情绪分型
- 场景化体验
- 角色成长反馈

而不是：
- 完整内容宇宙
- 重游戏玩法
- 多系统运营后台

## 5. 本项目统一术语

| 术语 | 含义 |
|---|---|
| `momo` | 主陪伴角色 |
| 情绪气象台 | 记录情绪类型、强度、一句话，并给出邀请式去向 |
| 解忧树洞 | AI 文本对话模块 |
| 光粒漂流 | 快乐模式 |
| 云团呼吸 | 低落模式 |
| 气泡释放 | 愤怒模式 |
| 解忧盲盒 Lite | 轻量安慰卡模块 |
| 我的 `momo` | 成长与当前状态页 |
| 结束反馈 | 用户完成聊天或模式后的帮助反馈 |
| 高危熔断 | 命中高危表达后进入安全链路 |

---

## 6. 不允许擅自扩展的范围

开发过程中以下内容视为冻结禁区：
- 不新增第四种情绪模式
- 不把三模式扩成关卡系统
- 不把盲盒扩成复杂内容池
- 不恢复专注森林
- 不恢复社区
- 不恢复塔罗 / MBTI / 星座
- 不接语音链路
- 不做商城和皮肤系统

---

## 7. 什么时候可以进入封测

满足以下条件后才进入封测：
- 首页 / 情绪气象台 / 树洞 / 三模式 / 盲盒 Lite / 我的 `momo` 可用
- 高危熔断可用
- 删除 / 举报 / 隐私 / 年龄门槛可用
- 埋点完整
- QA 文档中的 P0 用例通过

---

## 8. 文档维护规则

- 功能边界变更先改 `01_PRD`
- 流程名、模式名变更先改 `02_UX`
- 角色与视觉变更先改 `03_UI`
- API 或表结构变更先改 `05_Backend`
- Prompt 或高危词变更先改 `06_AI`
- 指标口径变更先改 `07_Analytics`
- 测试范围变更先改 `08_QA`
- 排期变更先改 `10_Sprint`
