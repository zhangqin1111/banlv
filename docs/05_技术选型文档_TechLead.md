## 1. 选型原则（先守后攻）

1. **稳定优先**：情绪疗愈场景崩溃成本 = 用户信任成本，选成熟而非最新。
2. **国产合规**：AI 模型 / 第三方服务必须完成国内合规备案。
3. **一码多端**：一次开发跑通 iOS / Android / 小程序（Web 为远期）。
4. **可灰度可降级**：AI / 游戏 / 动画都要有降级兜底。
5. **团队友好**：评估现有团队栈，避免冷门技术成为孤岛。

## 2. 客户端跨端方案对比

| 维度 | Flutter 3.x | React Native 0.74 | Taro (多端) | 原生双端 |
|------|-------------|-------------------|-------------|----------|
| 性能 | ★★★★★ 自绘引擎 | ★★★☆ Fabric 架构 | ★★ | ★★★★★ |
| 动画能力 | ★★★★★（Skia）| ★★★（Reanimated）| ★★ | ★★★★★ |
| 毛玻璃 / 粒子 | ★★★★ 支持 BackdropFilter | ★★★ 依赖原生库 | ★★ | ★★★★★ |
| 游戏引擎嵌入 | ★★★★ Platform View | ★★★ | ★★ | ★★★★★ |
| 小程序支持 | ✗ | ✗（Taro 可转）| ★★★★★ | ✗ |
| 包体增量 | 中（+8MB）| 中 | 小 | — |
| 热更新 | 受限 | Expo OTA | 原生 | 受限 |
| 团队招聘 | 友好 | 最友好 | 前端即可 | 成本高 |

**结论**：
- **iOS + Android 主 APP → Flutter 3.x**（治愈系动画多，Skia 有优势）
- **微信 / 抖音小程序 → Taro 4 + React**（另立 repo 共享设计 Token）
- **原生模块**：摄像头拍照、生物识别、Rive 动画、游戏引擎桥接用原生 Plugin

## 3. 游戏引擎对比（情绪气象台闯关模块）

| 引擎 | Cocos Creator 3.8 | Unity 2023 LTS | Laya 3 | 自研 Flutter 渲染 |
|------|-------------------|----------------|--------|-------------------|
| 包体 | ★★★★★ 最小（3–5MB）| ★★ (15MB+) | ★★★★ | ★★★★★ |
| 2D 性能 | ★★★★★ | ★★★★ | ★★★★ | ★★★ |
| 嵌入 Flutter | ★★★★ 成熟 PlatformView | ★★★（需封装） | ★★★★ | N/A |
| 学习成本 | 低 TS | 中 C# | 低 TS | 超高 |

**结论**：**Cocos Creator 3.8 + TypeScript**。理由：包体小、2D 闯关够用、可复用到小程序、团队易上手；三种模式各做一个 Scene，共用 InputSystem 与 Particle 资产。

## 4. 后端语言与框架

| 服务类型 | 选型 | 理由 |
|----------|------|------|
| 网关 / 聚合 | **Go + Gin / Kratos** | 高并发低延迟 |
| AI 网关 | **Python 3.12 + FastAPI** | 生态好、异步流式成熟 |
| 业务微服务 | **Go + Kratos**（统一）| 运维简单，Monorepo |
| 运营后台 | **Node.js + NestJS** | 前后端同构 |
| 实时通信 | **Go + Gorilla WebSocket / Socket.IO-Go** | SSE + WS 双通道 |

**不选**：
- ❌ Java / Spring（启动慢，团队小不必要）
- ❌ Rust（生态未稳，人才稀缺）
- ❌ PHP / Laravel（不适合 AI 场景）

## 5. AI 模型选型（合规优先）

### 5.1 LLM 对话模型（主用）
| 模型 | 中文质量 | 合规 | 成本（输入百万 token）|
|------|---------|------|-----------------------|
| **通义千问-Plus** | ★★★★★ | 已备案 | ¥4 |
| DeepSeek-V3 | ★★★★★ | 已备案 | ¥2 |
| 豆包-Pro-32k | ★★★★ | 已备案 | ¥2 |
| 智谱 GLM-4-Plus | ★★★★ | 已备案 | ¥5 |
| Claude / GPT-4o | ★★★★★ | ❌ 国内不可直接商用 | — |

**策略**：主用 **通义千问-Plus**（情感表达好 + 阿里云 SLA），备用 **DeepSeek-V3**（成本友好）。双 Provider 池动态路由。

### 5.1.1 三档路由模型（P0 成本规避方案）

为规避市场体验师报告中 P0-3 "LLM 成本吃毛利" 风险，对话请求按价值分档走不同模型：

| 路由档位 | 流量占比 | 使用模型 | 单次成本 | 触发规则 |
|---------|---------|---------|---------|---------|
| **Tier-0 端侧** | 40% | Qwen2.5-1.5B-Instruct (ONNX Mobile) | ¥0 | 问候 / 闲聊 / FAQ 关键词命中 |
| **Tier-1 轻量云** | 45% | Qwen-Turbo / DeepSeek-Lite | ¥0.003 | 普通树洞对话 / 情绪共情 |
| **Tier-2 旗舰云** | 15% | 通义 Plus / DeepSeek-V3 | ¥0.030 | 情绪日记生成 / 长上下文 / 高危边界 |

**降本辅助机制**：
- **语义缓存**：Embedding + Redis 相似度匹配，相似度 > 0.92 直接返回历史回答（命中率实测 ≈ 15%）。
- **上下文截断**：超过 4k token 自动摘要，避免无限增长。
- **免费档限流**：每日对话 10 轮，超出引导至盲盒机 / 小剧场。
- **情绪分类**：100% 端侧执行（ONNX 10MB 模型），不再调云端 API。

**新成本测算（DAU 50 万）**：
```
原月成本  ¥120,000
├─ Tier-0 替代: -¥48,000
├─ Tier-1 降级: -¥32,000
└─ 缓存命中:  -¥5,000
新月成本  ≈ ¥35,000
```

**灰度节奏**：
- Phase 1（MVP）：只开 Tier-1 + 限流，月成本目标 ≤ ¥50,000
- Phase 2（V1.1）：接入 Tier-0 端侧模型，A/B 验证满意度 ≥ 80% 后全量
- Phase 3（V1.2）：开启语义缓存 + 智能上下文截断

### 5.2 情绪分类模型（端云协同）
- **端侧**：微调 `bge-small-zh` 或 `ERNIE-tiny`（ONNX，10MB）
- **云端兜底**：`ERNIE 3.0 / 通义情感分类 API`
- **训练数据**：开源 NLPCC 情绪数据集 + 自有标注（首批 5000 条）

### 5.3 语音识别 / 合成
- ASR：**讯飞 / 阿里云** 语音流式 API
- TTS：**阿里云 CosyVoice**（情感 TTS，可做 momo 温柔声线）

### 5.4 内容安全
- **阿里云内容安全 LLM 防护** + **自研高危词库**（自伤 / 自杀 / 家暴）
- 高危触发走独立链路 `crisis-svc`，**不经过 LLM**

## 6. 数据库选型

| 场景 | 选型 | 说明 |
|------|------|------|
| 账户 / 订单 / 能量账本 | **PostgreSQL 16** | JSONB + 分区表 |
| 情绪日记 / 对话上下文 | **MongoDB 7** | 半结构化 |
| 情绪趋势 / 热力图 | **ClickHouse 24** | 列存聚合 |
| 缓存 / 会话 / 排行榜 | **Redis 7 Cluster** | — |
| 向量检索（长期记忆） | **Qdrant**（自建）或 **Milvus** | Qdrant 更轻 |
| 对象存储 | **阿里云 OSS** + 腾讯云 COS 灾备 | CDN |
| 全文搜索（内部运营） | **Meilisearch** | 轻量快 |

**不选 MySQL**：JSONB / 分区 / 生态上 PG 更优。

## 8. DevOps / 基础设施

- **容器 / 编排**：Docker + Kubernetes（阿里云 ACK）
- **服务治理**：Kratos + Nacos 配置中心
- **API 网关**：APISIX（限流、熔断、JWT）
- **CI/CD**：GitHub Actions + ArgoCD + Helm
- **监控**：Prometheus + Grafana + Loki + Sentry + SkyWalking
- **日志审计**：独立集群 + 写入只增追加，防篡改

## 10. Design System / 工具链

- 设计协作：**Figma**（Design Token 插件 → `tokens.json`）
- Token 流水线：`Style Dictionary` → Flutter ThemeData + Tailwind（Web）+ WXSS（小程序）
- 动画：**Rive**（momo 状态机）+ **Lottie**（粒子、庆祝）
- 图标：Figma → SVGO → Iconfont 统一分发

## 11. 安全选型

- 传输：TLS 1.3 + HSTS + 证书绑定
- 端侧加密：libsodium-jni / libsodium-ios
- 防抓包：Charles 检测 + SSL Pinning
- 防刷：设备指纹（数美 / 顶象）+ 行为验证码（极验）
- 审计：日志不可变存储（阿里云 SLS 归档）

## 12. 成本估算（DAU 50 万月）— 已应用 §5.1.1 降本

| 项目 | 原月成本 | 优化后 |
|------|---------|--------|
| 阿里云 K8s + 数据库 + OSS + CDN | ¥35,000 | ¥35,000 |
| LLM 调用（三档路由 + 端侧前置 + 缓存）| ¥120,000 | **¥35,000** |
| TTS/ASR 语音（沉默用户停止预生成）| ¥25,000 | ¥18,000 |
| 推送 / 短信 | ¥8,000 | ¥8,000 |
| 监控 / SaaS | ¥6,000 | ¥6,000 |
| **合计** | ¥194,000 | **≈ ¥102,000 / 月** |

按 5% 付费率 × ¥18 订阅 ≈ ¥45 万收入，毛利从原方案的 ¥26 万 → **¥35 万**，为人力与渠道投放留出空间。

**持续降本手段**：
- 用户沉默 30 天自动停止 TTS 预生成
- CDN 冷热分层 + Lottie 本地缓存
- 闲时调度（凌晨训练 + 报表）
- 三档模型按 A/B 结果动态调流量权重
