# EmoBot — 交互设计文档
> 角色：交互设计师 (UX) 视角  
> 版本：v1.0 · 2026-04-17

---

## 1. 交互设计原则（Emo 五则）

1. **Zero Pressure**：没有倒计时 / 失败惩罚 / 打卡断签红色提醒。
2. **Always Respond**：任何操作 100ms 内必有视觉或触觉反馈。
3. **One Thumb First**：主操作区控制在屏幕下半屏单手拇指热区内。
4. **Graceful Interrupt**：任意疗愈流程可一键暂停，状态不丢失。
5. **Empathic Feedback**：反馈文案以"你 / 我们"句式，不用"错误 / 失败"。

## 2. 全局信息架构 (IA)

```
APP
├─ 启动/登录/注册（沉浸浮层，非 Tab）
├─ 主 Tab 1 · 首页（岛屿 + 功能四入口）
│   ├─ 解忧树洞 → 对话页 / 历史页
│   ├─ 专注森林 → 养成页 / 装饰商店
│   ├─ 情绪气象台 → 打卡 / 日记 / 闯关游戏
│   └─ 解忧盲盒机 → 开盒 / 结果卡 / 历史
├─ 主 Tab 2 · 今日情绪（摘要 / 趋势）
├─ 主 Tab 3 · 我的（momo 成长 / 设置 / 订阅）
└─ 悬浮层 · 危机干预通道（全局浮标，高危词触发出现）
```

## 3. 全局导航逻辑

- **底部 3 Tab**（首页 / 今日情绪 / 我的）全局常驻。
- **顶部返回区** 只用手势左滑 + 显式左上角箭头（禁用右上角返回减少误操作）。
- **跨模块跳转** 使用共享元素过渡动画（momo 为锚点穿梭不同页面，避免"跳闪"）。
- **深层返回** 支持长按底部返回键一键回首页。

## 4. 核心流程状态机

### 4.1 情绪气象台打卡 → **邀请式**选择（核心 · 已按 P0 规避调整）

> **⚠️ 规范变更（v1.1）**：原"自动分流闯关"违背 Zero Pressure 原则，禁用。一律改为"先共情，再邀请"。

```
[输入情绪文本]
    ↓ 端侧 NLP 识别（不做自动跳转）
    ↓
[AI 共情回复 2 句]   ← momo 触手轻摆，柔和粒子聚集
    ↓
[浮出 3 张选择卡 · 横向毛玻璃]
    ├─ 「想继续说说」       → 留在树洞继续对话
    ├─ 「想安静写下来」     → 进入情绪日记
    └─ 「想让 momo 陪我玩一会」 → 进入匹配的情绪小剧场
    ↓ （高危词命中独立链路）
[高危熔断] → 柔光浮层 · 3 出口：
    ├─ 「我想自己待会」     → 静默守护模式 24h
    ├─ 「我想说说」         → 安全对话模式（仅安全模板）
    └─ 「拨打热线」         → 一键呼出
```

**文案与交互要求**：
- 禁止使用"已为您启动 X 模式 / 系统检测到您愤怒"等机械措辞。
- 选择卡 24h 内记住用户偏好：若用户连续 3 次选日记，则之后把日记置顶。
- 剧场入口默认**不在首页常驻**，只在用户主动打卡后才出现。
- 首次使用的新用户：前 3 天**不展示剧场卡**，让用户自然发现树洞 / 日记。

### 4.2 新用户首次 3 分钟流程
```
打开 APP → Splash（momo 泛光 1.2s）→ 登录页 → 手机号 → 验证码（自动填充）
→ 昵称 → 兴趣标签多选（跳过可）→ 首页加载（momo 浮现 + 岛屿淡入）
→ 引导手势（手指 icon 指向树洞）→ 关闭引导 → 自由探索
```

### 4.3 解忧树洞对话流
```
空页 → 键盘弹出 / 语音按钮长按
→ 发送 → 本地消息气泡即时出现（乐观 UI）
→ AI "思考中" 指示（momo 触手微摆动，非"..."三点，更温柔）
→ 流式输出气泡（逐字淡入，速度可设置慢速）
→ 三轮对话后，底部推荐快捷卡：呼吸练习 / 写日记 / 闯关一局
```

## 5. 手势与触控规范

| 手势 | 触发 | 用途 |
|------|------|------|
| 单击 | 任何按钮 | 主操作 |
| 长按 | momo / 卡片 | 弹出情绪菜单 |
| 双击 | 气泡 | 收藏到树洞金句 |
| 左滑 | 页面 | 返回上级 |
| 右滑 | 历史卡片 | 快速收藏 / 删除 |
| 摇一摇 | 盲盒机页 | 扭盒（带物理反馈）|
| 下拉 | 首页 | momo 变身查看 |

**闯关游戏操控**（参考蛋仔派对轻量化双区）：
- 左拇指：虚拟摇杆（可吸附 + 松手回弹）
- 右拇指：3 颗按键（跳跃 / 滚动 / 飞扑），只有愤怒模式启用三颗，其它两颗
- 首次进入闯关 5s 浮层手势教学，之后不再出现

## 6. 微交互规范 · Micro-interactions

### 6.1 情绪滑杆（0–10）
- 拖动时 momo 颜色跟随变化（蓝→紫→粉）
- 每跨越 2 格触发**一次 Haptic Light 震动**（iOS UIImpactFeedback.light）
- 松手后 300ms 内气泡出现，显示"你此刻大约是 X 分，对吗？"

### 6.2 扭蛋盲盒
- 长按：盲盒机摇晃预热
- 松手：3D 旋转 800ms + 木质齿轮咬合音效
- 卡片从盲盒口"吐出"并在屏幕中央放大（Elastic 缓动）

### 6.3 momo 交互
- 长按屏幕任意空白：momo 游到手指附近（Siri Ball 等效）
- 双击 momo：随机触发一个陪伴动画（12 套）
- 连续 7 天未打开：momo 透明度降到 40%，配合推送文案唤回

## 7. 反馈系统 · Feedback Matrix

| 操作 | 视觉 | 听觉 | 触觉 |
|------|------|------|------|
| 登录成功 | 粒子聚集 | 水珠音 | 中震 |
| 情绪打卡 | 环形图更新 | 气泡音 | 轻震 |
| 闯关完成剧场 | 光晕爆发 | ASMR 铃 | 双连震 |
| 击碎障碍 | 破裂粒子 | 脆响 | 节奏震 |
| 收到 AI 回复 | 触手摆动 | 近似"ding" | 无 |
| 高危熔断 | 浮层柔光 | 低频嗡音 | 无 |

## 8. 空状态 / 错误态设计

- **空状态**：不用插画大空，改用 momo 漂浮一句"今天还没有情绪被记录哦"
- **加载态**：momo 呼吸动画替代 Spinner
- **网络错误**：不用红字"失败"，改成"momo 暂时联系不到云朵，稍后再试？"
- **AI 超时**：5s 未响应显示 "momo 在认真想，要不要换个说法？"再等 10s 超时提示。

## 9. 情绪小剧场交互节奏设计（差异化 · 原"闯关"已更名）

> **术语统一**：文档全文 "通关→完成剧场"、"关卡→一段剧场"、"模式→剧场主题"。下表保留"快乐/低落/愤怒"作为内部开发代号，面向用户文案见《产品文档 §12.2》。


| 要素 | 快乐 | 低落 | 愤怒 |
|------|------|------|------|
| 开场载入 | 飞船上升 1.0s | 缓慢渐入 2.5s | 雷云拉入 1.5s |
| 背景音 | ASMR 铃 + 欢快短音 | 钢琴 + 雨声 | 白噪音 + 低频 |
| 失败提示 | 无（无淘汰） | 无（无淘汰） | 回到存档点 |
| 共情弹幕 | "你闪闪发光！" | "没关系慢慢来" | "情绪 -10%" |
| 结尾过渡 | 光瀑 + 勋章 | 冥想音频弹层 | 深呼吸引导 |

**关键节奏**：闯关开始到第一次有效操作**不超过 6s**，否则用户会走神；单关**控制 1–3 分钟**，超时自动"休息一下"浮层，避免沉迷。

## 10. 可达性与容错

- 所有点击目标 ≥ 44×44pt（Apple HIG / Material 共识）
- 语音输入降级：识别失败显示"我没听清，请再说一次"并自动录一次
- 防误触：三大关键危险按钮（删除日记 / 解绑手机 / 取消订阅）需二次确认 + 气泡文案
- 色盲模式：情绪状态必须文字 + 图标双冗余

## 11. 文案语气规范（UX Writing）

| 场景 | ❌ 避免 | ✅ 推荐 |
|------|--------|---------|
| 错误 | 网络错误 | momo 迷路啦，拉我一下 |
| 登录失败 | 密码错误 | 验证码好像不太一样呢 |
| 删除确认 | 确认删除？不可恢复！ | 真的要让这段记忆飘走吗？ |
| 空状态 | 暂无数据 | 等你写下第一段心情 |
| 完成剧场 | 恭喜完成剧场！ | 你的勇敢被 momo 看见了 |

## 12. 推送与唤回交互

- **推送密度**：单日 ≤ 2 条；不做打卡断签红色焦虑推送。
- **智能时机**：AI 基于用户过往打卡时段选最合适的 2 个时间窗。
- **内容**：第一人称 momo 口吻，如「我 bubble 了一个新梦想，想给你看」。
- **安静模式**：系统勿扰 / 用户设置 23:00–07:00 自动静默。

## 13. 可用性测试清单（上线前硬指标）

- [ ] 冷启动到首个可交互元素 ≤ 2.5s
- [ ] 气象台打卡 + 日记生成全程 ≤ 60s
- [ ] 闯关游戏崩溃率 < 0.3%
- [ ] 高危词识别误报率 < 5%，漏报率 < 0.5%
- [ ] 单手握持完成 95% 核心任务
- [ ] 用户首次使用 7 日后仍能独立找到"导出情绪日记"入口
