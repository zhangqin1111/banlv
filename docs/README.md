## 阅读顺序建议
1. 先读 **01 产品设计文档** 理解"要做什么、为什么做"。
2. 再读 **02 UI + 03 交互** 理解"长什么样、怎么用"。
3. 之后读 **04 架构 + 05 选型** 理解"怎么实现、用什么实现"。
4. 最后读 **06 市场体验师评审** 查漏补缺，形成上线前决策备忘录。

## 关键结论速览
- **差异化**：国产合规 AI + 水母 momo IP + 轻游戏化疗愈（非游戏）
- **技术主栈**：Flutter + Cocos + Go + Python + 通义/DeepSeek 双模型
- **核心风险 TOP 3**：AI 伦理 / 版号合规 / LLM 成本
