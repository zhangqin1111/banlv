# EmoBot 情绪疗愈 APP — 产品设计文档
> 角色：产品经理 (PM) 视角  
> 版本：v1.0 · 2026-04-17

---

## 1. 产品定位

**一句话定位**：EmoBot 是一款以「水母 momo 成长陪伴」为核心隐喻的 AI 情绪疗愈 APP，通过「AI 对话 + 情绪可视化 + 轻度闯关游戏 + 养成系统」四位一体，让用户在日常使用中完成自我觉察与情绪疏导。

**差异化定位（三句话）**：
1. 与 Moodnotes / Daylio 对比：不止于"打卡记录"，而是把情绪输入转化为**可交互的游戏剧情**。
2. 与小冰 / Replika 对比：把 AI 对话嵌入**具象化陪伴对象（水母 momo）**，而非纯文字 bot。
3. 与蛋仔派对 / 光遇 对比：游戏只是壳，**情绪疗愈**才是里，单局 1–3 分钟，轻陪伴而非重消耗。

## 2. 目标用户画像

| 画像 | Z 世代女大学生 | 职场高压白领 | 焦虑家长 |
|------|----------------|--------------|----------|
| 年龄 | 18–24 | 25–35 | 28–42 |
| 场景 | 睡前 / 课间 / 失恋 / 考试 | 通勤 / 午休 / 加班后 | 育儿空隙 / 夫妻冷战后 |
| 痛点 | 情绪波动大，羞于说出口 | 无法建立长期心理咨询关系 | 不愿被贴"有病"标签 |
| 付费意愿 | 对皮肤 / 装饰买单 | 对深度冥想 / AI 语音买单 | 对亲子情绪套装买单 |
| DAU 占比目标 | 55% | 30% | 15% |

## 3. 核心价值主张 (Value Proposition)

- **对用户**：低心理门槛的情绪出口 + 被看见的陪伴感 + 游戏化正反馈
- **对平台**：高粘性日活 + 情绪数据护城河 + 虚拟商品变现空间
- **对社会**：匿名情绪热力图反哺公共心理健康（远期 OKR）

## 4. 功能结构拆解（MoSCoW 分级）

### 4.1 Must Have（MVP 必做）
- 启动 / 注册 / 登录（手机号 + 短信）
- APP 首页俯视小岛 + 底部 Tab
- 解忧树洞（AI 文本对话，语音为 P1）
- 情绪气象台（晴/云/雨 + 0–10 滑杆 + 7 日环形图）
- 三大闯关模式中先上线**低落 + 愤怒模式**（快乐模式可作为 P1）
- momo 成长体系（三阶形态 + 情绪能量值）
- 个人中心 / 设置 / 隐私

### 4.2 Should Have（V1.1–V1.3）
- 专注森林（养成 + 装饰）
- 解忧盲盒机
- 快乐模式闯关
- 匿名情绪分享社区

### 4.3 Could Have（V2.0+）
- 星座 / 塔罗 / MBTI 拓展模块
- 城市情绪热力图
- 危机干预通道对接本地心理热线
- 多端同步（小程序 / 平板 / Web）

### 4.4 Won't Have（明确不做）
- 真人心理咨询师派单（规避医疗资质风险）
- 陌生人实时匹配聊天（规避社交风险）
- 打赏 / 大 R 抽卡型商业化（与疗愈调性冲突）

## 5. 核心用户旅程（3 条主路径）

### 5.1 新用户首次 7 日旅程
```
Day1 注册 → 兴趣标签 → momo 细胞幼体 → 首次气象台打卡（引导）→ 获得首份小确幸卡片
Day2 推送唤醒「momo 在想你」→ 解忧树洞首聊 → 连续对话奖励
Day3 首次触发闯关（随输入情绪分流低落/愤怒/快乐）→ 通关解锁皮肤
Day5 引导专注森林，浇水种出第一朵花
Day7 累计能量达标，momo 进化至二阶水母 → 首个里程碑勋章 → 引流到盲盒机
```

### 5.2 高压场景即时旅程（核心场景）
```
用户情绪崩溃 → 打开 APP → 首页 momo 主动靠近屏幕 → 一键进入解忧树洞
→ AI 识别高危词 → 温柔共情 + 推荐愤怒/低落闯关 → 1–3 分钟通关
→ 深呼吸引导 → 情绪日记生成 → 能量 +X → momo 安抚动画
```

### 5.3 低频唤回旅程
```
沉默 7 日 → 推送「你的水母 momo 变透明了」→ 进入 APP 触发"找回 momo"小剧情
→ 解锁补偿能量 → 引导至气象台 / 盲盒机
```

## 6. 情绪能量经济系统设计

### 6.1 能量来源与总量估算（日均上限约 60）
| 模块 | 单次获取 | 日封顶 | 触发要求 |
|------|----------|--------|----------|
| 解忧树洞 | 对话 +1 / 日记 +3 | 12 | 单轮≥10 字，避免灌水 |
| 气象台 | 打卡 +2 / 日记 +2 | 10 | 每日最多 3 条 |
| 闯关游戏 | 每关 5 / 障碍 1 | 30 | 避免沉迷 |
| 盲盒机 | 开盒 +3 / 打卡 +2 | 8 | 每日 3 次 |
| 拓展功能 | 首次 +3 | 3 | — |

### 6.2 能量消耗路径（3 类）
- **形态进化**：细胞 3000 → 水母 20000 → 终极形态（健康消耗）
- **装饰解锁**：999 款装饰 50–500 点不等（变现入口）
- **动画动作包**：安抚 / 欢快 / 温柔陪伴包各 1500 点（情感绑定）

### 6.3 防沉迷与伦理约束
- 单日闯关时长软上限 30 分钟（到达柔和提示而非强制）
- 情绪能量**不参与任何排行榜**
- 高危词检测触发熔断（见第 9 节）

## 8. 关键数据指标 (North-Star + Guardrail)

- **北极星**：7 日情绪记录留存率（目标 ≥ 35%）
- **核心指标**：
  - DAU / MAU = 30%+（陪伴类产品高于行业均值）
  - 人均日使用时长 12–18 分钟（不宜更长，违背疗愈初衷）
  - momo 二阶进化转化率 ≥ 40%
- **护栏指标**：
  - 负面情绪入口后 24h 内再次登录率（反推疗愈效果）
  - 高危词触发后成功跳转心理热线比例 ≥ 80%

## 9. 伦理与合规红线

1. **不做医疗诊断**，所有文案避免"抑郁""焦虑症"等临床词。
2. **数据最小化原则**：情绪日记默认**端侧加密**，云端仅存哈希。
3. **未成年人保护**：注册年龄 <14 必须监护人模式；<18 关闭深夜推送。
4. **AI 回复安全网**：内容过滤 + 危机识别 + 人工兜底审核。
5. **隐私合规**：符合《个人信息保护法》《生成式 AI 服务管理办法》《未保法》，取得 ICP + 网信办生成式 AI 备案。

## 12. P0 风险规避专章（上线前必做）

> 本章回应市场体验师评审报告中的 4 项 P0 风险，为 PM 侧决策备忘。技术细节分别落在《技术架构文档》§6 与《技术选型文档》§5。

### 12.1 AI 伦理红线 — 四层防护网

| 防护层 | 机制 | 责任方 |
|--------|------|--------|
| 输入层 | 自建高危词库（约 2000 词：自伤 / 自杀 / 家暴 / 未成年风险）+ 阿里云内容安全双校验；命中直接跳过 LLM，走人工写好的安抚模板 + 心理热线浮层 | crisis-svc |
| Prompt 层 | System Prompt 强约束：禁止"抑郁症/焦虑症/治愈"等临床词，强制"我不是医生"声明 | ai-gateway |
| 输出层 | 小模型裁判打分 < 0.7 丢弃并重试；每日 0.1% 人工抽检 | 安全运营 |
| 兜底层 | 任何置信度不足场景回复"建议与专业的人聊聊"+ 一键呼出热线 | 产品 |

- **前置投入**：约 ¥15 万（语料标注 + 裁判模型）+ 1 名全职安全审核 (¥15k/月)
- **残余风险**：长尾 case 仍可能漏检 → 每月 Red-Team 回归一次
- **上线闸门**：人工审核 500 条高危模拟对话误放率 = 0 才允许发版

### 12.2 版号 / 合规 — 把"游戏"重塑为"情绪剧场"

**去游戏化改造清单**（产品 + 文案同步执行）：
| 原词 | 新词 | 理由 |
|------|------|------|
| 关卡 / 99 关 | 情绪小剧场（每次随情绪生成）| 去竞技化 |
| 通关 / 胜利 | 这一刻被收藏 / 情绪被看见 | 去竞技化 |
| 快乐/低落/愤怒模式 | 阳光小剧场 / 云朵漫步 / 情绪风暴 | 去模式化 |
| 皮肤 | 陪伴形象 | 避开游戏概念 |
| 情绪能量 | 陪伴值（仅能换 momo 形态）| 去虚拟货币化 |
| 勋章 | 回忆册页 | 去成就化 |

**经济系统调整**：
- 陪伴值**只用于 momo 形态进化**，不能兑换装饰 / 皮肤。
- 装饰 / 陪伴形象走**独立内购**，不经过"虚拟货币"通道。
- 删除 99 关递进表述，改为"根据当下情绪动态生成一段 60–120s 的小剧场"。

**前置动作**：
- 上线前委托版号代理做合规预评估（约 ¥3–5 万）。
- 应用市场类目一律走 "**健康 / 生活方式**"，小米 / 华为 / 应用宝单独申请。
- 准备法律意见书一份备查。

**残余风险**：各地监管口径不一 → 对不同应用市场版本做微调（可通过 remote-config 下发）。

### 12.3 LLM 成本 — 三档模型路由 + 端侧前置

| 场景占比 | 使用模型 | 单次成本 |
|---------|----------|---------|
| 40% 问候 / 闲聊 / 关键词回复 | 端侧 Qwen2.5-1.5B（ONNX Mobile）| ¥0 |
| 45% 常规树洞对话 | Qwen-Turbo / DeepSeek-Lite | ¥0.003 |
| 15% 情绪日记 / 深度对话 / 高危场景 | 通义 Plus / DeepSeek-V3 | ¥0.030 |

**其他降本**：
- 语义缓存（常见问答命中率 ≈ 15%）
- 免费用户每日 10 轮树洞对话，超出引导至盲盒 / 小剧场
- 情绪分类 100% 端侧执行，不调云 API

**新成本测算**（DAU 50 万）：
| 项目 | 原月成本 | 降本后 |
|------|---------|--------|
| LLM 调用 | ¥120,000 | **¥35,000** |
| 其他 | ¥74,000 | ¥74,000 |
| **合计** | ¥194,000 | **¥109,000** |

按 5% 付费率 + ¥18 订阅 = ¥45 万收入，毛利从亏损转为约 ¥34 万 / 月。

**上线节奏**：
- MVP 阶段先靠 "限流 + 缓存" 把月成本压到 ¥50,000 以内
- V1.1 上端侧小模型彻底替代 40% 请求
- 端侧模型用户满意度需 A/B 验证 ≥ 80% 才允许全量

### 12.4 强制分流闯关 — 改为"邀请式选择卡"

**原流程（禁用）**：
```
输入情绪 → NLP 识别 → 自动跳转对应闯关模式
```

**新流程（上线）**：
```
输入情绪 → AI 共情回复 2 句 → 浮出 3 张选择卡
   ├─ 「想继续说说」 → 留在树洞
   ├─ 「想安静写下来」 → 进入情绪日记
   └─ 「想来个小剧场」 → 进入匹配的情绪剧场
```

**配套规范**：
- 首次使用引导**默认不推剧场**，用户自然发现。
- 推剧场文案改为柔性："要不要让 momo 陪你玩一会？"，禁用"已为您启动 X 模式"。
- 首页不常驻入口，仅在用户主动打卡后出现。
