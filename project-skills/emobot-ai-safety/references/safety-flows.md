# Safety Flows

Use this file when working on high-risk behavior.

## High-risk handling

1. Detect risky input before normal reply generation.
2. Stop ordinary Treehole flow.
3. Return a calm support template.
4. Offer immediate help-seeking guidance.
5. Record a safety event.
6. Do not continue roleplay or immersive chat in the same unsafe tone.

## Safety buckets

- self-harm or suicide intent
- harm to others
- abuse or acute danger
- requests for dangerous step-by-step advice

## Product rules

- Mood Weather can still save the check-in, but should not send the user into a normal mode scene if safety is triggered.
- Report and delete flows must remain reachable.
- Safety copy must not sound robotic or scolding.
