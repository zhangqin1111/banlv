# Release Playbook

Use this file when deciding whether a build should ship.

## First release posture

- one platform first
- invite-only beta first
- small cohort first

## Ship blockers

- broken safety flow
- broken delete or report flow
- broken mode-scene persistence
- broken growth state
- major streaming failure in Treehole

## Expand only when

- activation is healthy enough
- helpfulness score is acceptable
- no severe safety regressions appear
- AI cost stays within the target budget
